# Board-Man OSS Publication Checklist

## Do not publish until all are checked

### Security / privacy

- [ ] No `.env` files
- [ ] No tokens, API keys, cookies, or secrets
- [ ] No signing certificates
- [ ] No provisioning profiles
- [ ] No private bundle identifiers if they should remain private
- [ ] No personal absolute paths such as `/Users/yourname/...`
- [ ] No private logs
- [ ] No customer/user data
- [ ] No Discord webhook URLs
- [ ] No production deployment scripts
- [ ] No unnecessary build artifacts

### License / attribution

- [ ] Upstream Clipy license confirmed
- [ ] Original license text preserved
- [ ] Required copyright notices preserved
- [ ] Modified derivative status clearly stated
- [ ] `ATTRIBUTION.md` completed
- [ ] `LICENSE` finalized

### Repository quality

- [ ] Public repo name decided: `boardman` or `board-man`
- [ ] README explains what the tool does in 30 seconds
- [ ] Build steps are reproducible
- [ ] Screenshot or demo GIF added
- [ ] v0.1.0 release notes prepared
- [ ] Known limitations documented
- [ ] GitHub topics added
- [ ] Issue template added if needed

### Codex for OSS application readiness

- [ ] Public URL ready
- [ ] README ready
- [ ] LICENSE ready
- [ ] Release or tag created
- [ ] Short application text prepared
- [ ] Board-Man positioned as a macOS clipboard productivity OSS, not just a private fork
